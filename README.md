Голосовой агент, который звонит каждому новому потенциальному клиенту (n8n + Vapi) 
Что вы создаете 
Голосовой оператор по отбору потенциальных клиентов, который звонит новым потенциальным клиентам сразу после отправки ими формы, задает несколько отборочных вопросов, а затем записывает результаты в Google Sheets. 
Основной поток:
1. Потенциальный клиент отправляет форму (в демо-версии: триггер «Форма» в n8n).
2. n8n нормализует номер телефона.
3. n8n звонит потенциальному клиенту с помощью Vapi (Create Call).
4. n8n опрашивает Vapi до тех пор, пока статус звонка не изменится на «завершен» (Get Call).
5. n8n проверяет, не был ли это голосовая почта.
6. n8n записывает структурированные данные (поля квалификации) в Google Sheets.


Необходимые инструменты 
● n8n (логика рабочих процессов + HTTP-запросы)
● Vapi (голосовой агент + исходящие звонки)
● Google Sheets (журнал + просмотр pipeline)
● Дополнительно: номер Twilio (рекомендуется, если вы хотите увеличить объем исходящих звонков сверх суточных ограничений Vapi)





Данные, которые вы собираете 
1) Из формы (исходные данные) 
Минимальный набор полей, используемых в демо-версии: 
● Имя
● Номер телефона
● Электронная почта
● Название компании
● Должность
● Запрос (в чем требуется помощь)
● Размер компании

2) Из вызова (дополнительные данные о квалификации) 
Данные из структурированных выводов Vapi были извлечены и занесены в таблицу: 
● Интерес к услуге
● Мотивация
● Срочность / сроки
● Предыдущий опыт
● Бюджет
● Готовность к оплате (готовность оплатить предварительную оценку объема работ)
● Статус (завершено, голосовая почта, недействительный номер и т. д.)

Схема высокого уровня (скопируйте её перед началом разработки)
Триггер: отправка формы 
Шаг 1: Нормализация данных 
● Приведите номер телефона к стандартному 10-значному формату.
● Если номер недействителен: отметьте его как неверный формат и зафиксируйте это в журнале. 

Шаг 2: Позвоните клиенту (Vapi) 

● Создать вызов с помощью:
○ Идентификатор помощника
○ Идентификатор номера телефона (идентификатор вызывающего абонента)
○ Номер телефона потенциального клиента
○ Динамические переменные, передаваемые из промпта помощника

Шаг 3: Проверка статуса вызова 
● Подождать (например: 60 секунд)
● Получить сведения о вызове

● Если статус != завершен:
- Подождать (например: 10 секунд)
- Повторно получить сведения о вызове
● Повторять, пока status не станет = ended 

Шаг 4: Обработка голосовых сообщений и отвеченных звонков 

● Если endedReason == голосовая почта:
○ Зарегистрировать статус голосовой почты
● В противном случае:
○ Извлечь структурированные данные и зафиксировать всё в журнале


Разбор рабочего процесса n8n (по этапам) 
1) Триггер формы (демо) 
● Используйте триггер n8n Form Trigger, чтобы пользователи могли создавать копии с помощью шаблона.
● В рабочей среде это, как правило, будет веб-хук, получающий данные из формы на вашем веб-сайте.


2) Code Node: Нормализация номера телефона 
Цель: вывести 10-значный телефонный номер (без скобок, тире и пробелов) или вывести incorrect format. 
Использованные логические правила: 
● Удалить нецифровые символы. 
● Если номер начинается с кода страны, например 1, удалите его (предполагается, что речь идет о США). 
● Если окончательное количество цифр не равно 10, вернуть incorrect format. 
Совет из видео: 
● Скопируйте полученный JSON из n8n и вставьте его в Claude (или другой виртуальный помощник) с одноразовым запросом, чтобы сгенерировать узел кода. 
3) Узел IF: Неверный номер телефона? 
Условия: 
● phone_number == "incorrect format" 
True branch: 
● Заносит потенциального клиента в таблицу со статусом «Недействительный телефон». 
False branch: 
● Продолжает звонить потенциальному клиенту.



4) HTTP-запрос: «Call Lead» (Vapi Create Call) 
Назначение: инициирует исходящий вызов. 
Method: POST 
Auth: Generic Credential, Bearer Auth 
● Создайте учетные данные для Vapi в n8n.
● Вставьте свой закрытый API-ключ Vapi.
Key body fields (conceptual): 
● assistantId: ваш Vapi ассистент 
● phoneNumberId: номер телефона Vapi (или импортированный номер Twilio) 
● customer.number: номер телефона потенциального клиента (formatted as +1 + 10 digits) 
● assistantOverrides.variableValues: динамические переменные для промпта помощника 
Динамические переменные («магия»): 
Эти переменные отправляются из n8n и вставляются в командный промпт системы Vapi: 
● lead_name 
● lead_company_name 
● Lead_request


5) Узел ожидания (начальное ожидание) 
● Пример: 60 секунд
● Это дает время на:
- Звонок должен раздаться
- Разговор должен состояться
6) HTTP-запрос: получение сведений о вызове (Vapi Get Call) 
Цель: получить статус и результаты вызова (включая структурированные данные). 
Method: GET 
URL pattern: 
● Идентификатор вызова получается из ответа на запрос «Create Call».
● Добавьте его к конечной точке (в документации указан путь вида /call/{id}). 
ВАЖНО: 
● В видео Vapi иногда возвращал много элементов (вероятно, это была ошибка). В рабочем процессе в качестве «источника достоверной информации» используется только первый элемент.




7) IF Node: Статус завершен? 
Условия: 
● status == "ended" 
True branch: 
● Продолжить проверку голосовой почты и ведение журнала. 
False branch: 
● Подождите (например, 10 секунд), а затем вернитесь к пункту «Получить сведения о вызове». 
Этот цикл представляет собой систему опроса. 
8) IF Node: Проверка голосовой почты 
Условия: 
● endedReason == "voicemail" 
True branch (voicemail): 
● Запишите статус голосового сообщения в таблицу (чтобы потом перезвонить). 
False branch (answered): 
● Извлекать структурированные результаты и регистрировать полные данные о квалификации вызовов. 
9) Google Sheets: Запись результатов 
Создайте столбцы для обоих: 
● Поля для отправки форм
● Структурированные выводы Vapi
В демонстрационной версии структурированные данные отображаются в столбцах (например: мотивация, срочность, бюджет, намерение и т. д.).

Vapi настройка (помощника) 
Model 
● Provider: OpenAI 
● Model: GPT-4o (как показано в видео) 
Первое сообщение 
● Настройте помощника так, чтобы он ждал, пока пользователь заговорит первым (это выглядит более естественно, когда кто-то отвечает на звонок). 
Системный промпт (структура) 
Включает: 
1. Identity: Кем является помощник (например: «Эллиот, специалист по отбору потенциальных клиентов по исходящим звонкам в Upit»).
 2. Рекомендации по стилю и ответам: лаконично, естественно, профессионально. 
3. Prospect variables: для динамических значений используйте двойные фигурные скобки. 
○ {{lead_name}} 
○ {{lead_company_name}} 
○ {{lead_request}} 
4. Ход беседы + обязательные темы: 
○ Подтверждение заинтересованности
○ Мотивация
○ Срочность / сроки
○ Предыдущий опыт
○ Бюджет
○ Намерение совершить покупку (отсеивание тех, кто просто присматривается)

5. Крайние случаи:
○ Неверный номер
○ Неподходящее время
○ Пользователь просит прекратить или не звонить

6. Поведение при завершении вызова: включить встроенную функцию «завершения вызова». 
Примечание по этике, использованное в видео: 
● Помощник представляется как ИИ агент.


Структурированные выводы Vapi (как получить чистые поля) 
Структурированные результаты заменяют устаревшие схемы подведения итогов и оценки успешности.
Создавайте структурированные результаты для тех полей, которые именно вы хотите получить, например:
● Мотивация (строка)
● Срочность (строка)
● Предыдущий опыт (строка)
● Бюджет (строка)
● Интерес к услуге (строка)
● Намерение совершить покупку (логическое значение или строка)
● Статус (строка)
Как настроить: 
1. В Vapi откройте раздел «Structured Outputs».
2. Добавьте новый элемент (выберите тип, например string/boolean).
3. Напишите чёткое описание того, что нужно извлечь.
4. Свяжите его с нужным помощником. 

Где это встречается в диалоге: 
● Внутри вложенного объекта, который часто обозначается как  artifacts → structuredOutputs. 



Распространенные ошибки (и способы их устранения)
Неправильное форматирование номера телефона приводит к сбоям при звонках 
Исправление: 
● Преобразовать в число, состоящее из 10 цифр. 
● Если звонок предназначен только для США, добавьте +1 в запрос на звонок. 
● Перенаправлять недействительные номера в ветвь регистрации. 
Информация о звонках становится доступна не сразу 
Исправление: 
● Добавить опрос: : 
○ Подождите 60 секунд 
○ Получить звонок 
○ Если процесс не завершился, подождите 10 секунд и повторите


Ограничения на исходящие звонки с номеров, приобретенных в Vapi 
Исправление: 
● Импортируйте свой собственный номер Twilio, если вам требуется увеличить объем исходящих звонков. 
В ответе «трудно найти» структурированные данные JSON 
Исправление: 
● Изучайте ответ, пока не найдете artifacts потом structuredOutputs. 

Контрольный список для тестирования 
● Отправить форму с указанием действующего номера телефона в США.
● Убедиться, что звонок инициирован.
● Убедиться, что цикл опроса прекращается, когда статус меняется на «завершено».
● Протестировать сценарий работы голосовой почты и убедиться, что маршрутизация осуществляется правильно.
● Убедиться, что в Google Sheets записана строка со следующими данными:
- Исходные данные
- Структурированные результаты
- Итоговый статус
● Проверить нестандартные форматы телефонных номеров (9 цифр, лишние цифры, крайние случаи с кодами стран). 

Контрольный список для быстрой настройки (для зрителей) 
1. Получите ключ API Vapi и сохраните его в n8n в качестве учетных данных для auth Bearer. 
2. Создайте и настройте Vapi Assistant (системные подсказки + структурированные результаты). 
3. Подключите номер телефона в Vapi (номер Vapi или импортированный номер Twilio). 
4. Обновите узел n8n «Call Lead» с помощью: 
○ Assistant ID 
○ номер телефона ID 
5. Скопируйте  Google Sheet шаблон подключите его к workflow. 
6. Отправьте тестовую форму и убедитесь, что данные заносятся в таблицу. 
Идеи по расширению workflow
● Добавить повторные попытки в случае отсутствия ответа (позвонить позже).
● Отправить SMS-сообщение, если включена голосовая почта.
● Автоматически создать запись о потенциальном клиенте в CRM (HubSpot, Pipedrive, Airtable).
● Автоматически создать ссылку на встречу в календаре, если контакт соответствует критериям.
● Добавить цикл анализа звонков и оптимизации подсказок (анализировать транскрипты, со временем улучшать подсказки).
